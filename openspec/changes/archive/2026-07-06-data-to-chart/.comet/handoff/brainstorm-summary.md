# Brainstorm Summary

- Change: data-to-chart

## 确认的技术方案

### SKILL.md 结构
指令式主入口（SKILL.md）+ 3个参考文件（reference/），部署于 `~/.claude/skills/data-to-chart/`。

5步执行流程：
1. 场景分析：读取 chart-mapping.md → 匹配业务关键词 → 确定图表类型
2. 数据获取：项目文件 → Web搜索多轮 → 示例数据（降级链）
3. 数据整理：解析为 ECharts 可用格式，适应性处理
4. 双轴验证：数据结构 vs 业务场景，冲突时业务优先
5. 代码生成：参考模板灵活适配 → 完整 HTML（含选择理由说明）

### 触发方式
- 宽泛关键词触发（画图/图表/可视化/展示数据等）
- 斜杠命令：`/data-to-chart`
- 首版仅 Claude Code

### 模板策略
适应性优先，参考模板不硬性套用。数据有什么就展示什么。

### 地图处理
支持中国地图 CDN JSON，其他地区提示需注册。

### 数据搜索
多轮搜索，必要时打开1-2个结果页面提取详细数据。

## 关键取舍与风险

- 一致性弱但适应性强：不强制固定模板，靠规则保证选型正确
- 地图仅覆盖中国场景，其他地域要提示
- 搜索数据可能不准确但不阻塞输出

## 测试策略

5条端到端验证（折线图、堆叠柱状图、CSV解析、冲突场景、无数据搜索）
